# ETF 策略里的截面思维（聚宽社区方法论综述）

> 来源：[聚宽社区](https://www.joinquant.com/view/community/detail/2e4a0cb7700c51114e3cf4e4f39b8173?type=1) | 2026-05-07 通过用户手动复制粘贴 ingest

## 一句话定位

**少有的"方法论级"ETF 综述**：不讲单个指标 / 单个策略，而是把社区里散落的 ETF 策略按**4 层架构（候选池 → 横向打分 → 组合构建 → 择时风控）+ 5 类截面做法（动量排序 / 趋势确认 / 低相关组合 / 权重优化 / 候选池构建）** 串成完整脉络。**直接对接 [[ETF轮动与交易策略]] 主题**，把它从"策略集合"升级为"框架性主题"。

## 核心命题

| 维度 | 时间序列思维 | **截面思维（本文核心）** |
|---|---|---|
| 关注 | 单 ETF 强不强 | 一篮子 ETF 谁更强、谁同质、怎么配 |
| 工具 | 均线 / MACD / [[RSRS]] / BOLL | 横向打分 / 排序 / 相关性 / 权重 |
| 核心问题 | "现在该不该买" | **"该选谁、怎么配、哪些其实长得一模一样"** |
| 成熟形式 | — | **截面 + 时间序列混合体** |

## 4 层架构（最重要的洞察）

```
[1] 候选池      ← 入口净化（成立时间 / 流动性 / 同质化 / 相关性去重）
       ↓
[2] 横向打分    ← 选谁（动量 / 趋势 / 行业宽度 / 相对强弱）
       ↓
[3] 组合构建    ← 怎么配（Top N → 低相关约束 → 权重优化）
       ↓
[4] 择时 + 风控 ← 何时收手（大环境过滤 / 标的失效退出，避免过滤器叠加）
```

**作者最重要的建议**：
> "**先别急着调因子，先把池子做干净。**"

## 5 类典型截面做法

### 1. [[ETF动量排序]]
- 池中每只 ETF 算动量分（N 日累计收益 / 回归斜率 / 年化×R² / 乖离 / 加权趋势分）
- 横向排序 → 买 Top 1 / Top N
- **副作用**：换手率高（不是趋势没了而是排名变了）

### 2. [[趋势确认过滤]]（排序后做趋势确认）
- 解决"虽然在批中最强，但买进去是不是太晚"
- 工具：[[RSRS]] / 均线站上 / BBI / BOLL / 量价异常 / 日内大跌排除
- 结构变化：先算分 → **再确认** → 通过才上车

### 3. [[低相关组合]]（本文最有感触的一类）
- 问题转变："谁最猛？" → **"谁和谁别太像？"**
- 做法：剔除过同步标的 / 趋势筛选后选低相关组 / 一组里再做 Top N
- ⇒ 真正像"组合"而非"追涨切换器"

### 4. [[权重优化]]
- 等权 / 波动率倒数 / [[风险平价]] / 协方差优化 / EPO
- 难点：**协方差估计不稳 + 样本内 vs 样本外变形 + 权重日调高换手**
- 建议：先把候选池 + 低相关 + Top N 做扎实

### 5. [[ETF候选池构建]]（最易被忽略但最关键）
- 过滤短成立期 / 低流动性 / 高同质化
- **聚类**走势相似的保留代表
- 相关系数阈值剔除重复
- 按资产类别 / 主题 / 跨境属性分组

## ETF 策略演化路径（决策层次的成熟）

```
v1: "找最强的一个"
       ↓
v2: "找最强的几个，再确认趋势"
       ↓
v3: "找一组更强、彼此更不雷同、权重更合理的 ETF"
       ↓
v4: "先把 ETF 池做干净，再做横向打分，再做组合构建，再做择时和风控"
```

⇒ **变成熟不是因为指标调到最优，而是决策层次变完整。**
