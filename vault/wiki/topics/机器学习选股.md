# 机器学习选股

> 用机器学习模型替代或增强传统因子线性合成的选股范式

## 主题定位

随着传统多因子模型 IC 衰减加快（OpenAlphas 给出"超过 30%"的衰减率），**机器学习选股**成为获取增量 alpha 的关键路径。本主题汇总两条主线：

1. **端到端预测** — 用模型直接学映射 X → y，不显式输出可解释因子。代表：[[LightGBM]] + [[贝叶斯超参优化]]
2. **自动因子挖掘** — 用算法自动生成可解释因子，再用传统多因子框架做组合。代表：[[AlphaZero因子挖掘]] / [[AutoML-Zero]]

## 关键素材

- [[2026-05-05-openalphas-lightgbm-bayesian]]：S 档；端到端 LightGBM + Optuna 50 轮贝叶斯超参，年化 38.7% / MDD 12.3%
- [[2026-05-05-openalphas-alphazero-factor-mining]]：A 档；图状程序表达式 + 关联变异 + 退化变异，3 个核心因子 ICIR > 7

## 核心概念

### 模型层
- [[LightGBM]] / [[GBDT]] — 梯度提升决策树家族
- [[AlphaZero因子挖掘]] / [[AutoML-Zero]] — 进化算法 + 程序合成

### 工程纪律
- [[时序划分]] — 防数据泄露的硬约束
- [[贝叶斯超参优化]] / [[Optuna]] — 高效超参搜索
- [[3σ去极值]] / [[Z-score标准化]] — 数据预处理

### 评价指标
- [[ICIR]] / [[斯皮尔曼秩相关系数]] — 因子稳定性
- [[IC选股策略]] — 截面排序 Top-N 等权范式

## 端到端 vs 自动因子的权衡

| 维度 | 端到端（LightGBM） | 自动因子（AlphaZero） |
|---|---|---|
| 解释性 | 弱（黑箱） | **强**（图状程序可读） |
| 训练成本 | 中 | **高**（进化迭代） |
| 因子产出 | 无 | **可与传统因子库合并** |
| 过拟合控制 | 贝叶斯超参 + 时序划分 | 长度约束 + 退化变异 |
| 适合场景 | 快速产出预测信号 | 深耕因子库 |

## 与传统多因子的关系

- 传统：人工因子 → 线性合成 → IC 选股
- 机器学习：（自动因子 / 端到端）→ 模型预测 → IC 选股
- 评价指标体系是**复用的**（[[ICIR]] / [[IC选股策略]]）

## 与可转债定价模型的对比

[[可转债定价模型]] 走的是**机理驱动**路径（BS / ZL 模型），用 [[错误定价因子]] 作为另类 alpha。
本主题走的是**数据驱动**路径，用 ML 自动捕捉非线性。两者**完全互补**：
- 信用强股 / 牛市风险偏好阶段 → 数据驱动占优
- 信用弱 / 熊市保守阶段 → 机理驱动占优

## 优化方向

- 集成学习：LightGBM + 自动因子混合
- Walk-forward 滚动训练（更严格的 OOS 验证）
- Stacking：底层多模型 + 上层 LightGBM meta
- 引入另类数据（专利、舆情、宏观）扩展因子来源
