# 动量衰竭早期识别

> 用三种**异源**信号在不同维度上提早识别趋势/动量行将结束 — 解决"日频动量策略在持续趋势市能赚但在动量拐点附近吐回收益"的核心痛点

## 主题定位

动量策略的根本风险**不是没有信号**，而是**入场容易、止盈难**。日频 R²-加权动量、双窗口 confluence、STR 反转 tilt 等机制都是"如何选最强标的"的问题；本主题聚焦的是另一个问题：**当我已持有趋势冠军时，怎么早一步知道它要崩**。

来源痛点：[[ETF_New]] 项目 2025 复盘发现，dual+STR composite 在 93% risk_on 的趋势市中累积让出 1pp Sharpe — 单纯靠价格的 STR_z 反转作为止盈触发**反应太慢且会误伤健康趋势**。需要更早、更可靠的衰竭信号。

## 四层衰竭检测框架（2026-05-08 升级）

| 层级 | 工具 | 数据要求 | 信号性质 | 滞后性 | 触发频率 |
|---|---|---|---|---|---|
| **价格结构层** | [[九转序列]] | OHLC 日频 | 离散：连续未创新高/低计数到 N | 中 | 稀疏 |
| **趋势导数层** | [[低延时趋势]] | 收盘价 | 连续：频域低频成分斜率 | 低 | 中等 |
| **微观结构层** | [[VPIN]] / **[[VWPIN]]** | 分钟级量价 + 主买/卖笔数 | 连续：订单流毒性分位 | **最低** | 高（连续监测） |
| **行为金融层**（new） | [[信息离散度ID]] / [[BIAS减仓]] | 日频 OHLC | 连续：涨法分布画像 | 中（12 月窗口） | 月频 |

**行为金融层补充**（[[2026-05-08-earletf-fip-bias-trim]]）：
- ID 高（暴涨集中）= [[FIP效应]] 离散信息组 = 6 月动量 2.91% / 仅持续 3 月
- ID 低（慢涨累积）= 连续信息组 = 6 月动量 8.86% / 持续 8 月不反转
- BIAS > 15% / 20% → ID 高的事后印证 → 触发减仓
- 与微观结构层互补：VPIN 看**资金性质**，ID 看**信息到达方式**

## 主题升级（2026-05-31 追加：策略元风险层）

[[2026-05-31-dbjg-fabozzi-strategy-decay-risk]] 引入 [[Frank Fabozzi]] 论文《Measuring Strategy-Decay Risk》，把"衰竭研究"从单标的层**提升到整个策略的元风险层**：

```
[Layer 0] 策略元层（new）
          ↓
  Decay-Risk Frontier (Sharpe × MRP)
  → 衡量整个策略在不同 regime 下的寿命
          ↓
[Layer 1-4] 单标的层（已有）
  → 行为金融 / 微观结构 / 价格结构 / SMC 等
```

**核心新概念**：
- [[MRP最小Regime表现]]：MRP = Min(Sharpe across regimes)
- [[衰减风险前沿]]：类比 Markowitz，但优化"收益 × 寿命"
- [[Alpha半衰期]]：策略衰减到一半所需时间
- [[策略耐久性]]：研究焦点从"发现 Alpha"→ "延长 Alpha 寿命"

**实证警示**：经典因子在历史 Sharpe 上漂亮但 MRP 可能极差：

| 因子 | Sharpe | MRP |
|---|---|---|
| Debt Issuance | 1.14 | **-0.05** |
| Investment | 0.45 | **-1.01** |
| Size | 0.16 | **-0.93** |

→ **高 Sharpe ≠ 高耐久**。这与 V25 v5i 复盘"IS Sharpe 漂亮 ≠ OOS 可交付" 的工程经验**完美对应**。<!-- confidence: INFERRED -->

⚠ **重要警示**：[[接受规则]] 来自外汇背景，A 股因涨跌停 + T+1 制度，**直接迁移需重大改造**（参考 [[均线偏离度]] 的"20 日中至少 10 日"双重过滤可视为本土等价物）。

**关于微观结构层工具选择**：
- **学术理论**：[[VPIN]]（Easley-López de Prado-O'Hara 2012）— 等成交量分桶 + [[BVC算法]]
- **A 股工程实践**：[[VWPIN]]（李平等 2020 / 广发 2022 因子化）— 等时间区间 + 订单笔数不平衡

**A 股直接证据**：广发证券 2022 [[2022-02-21-gf-hfdata-factor-series-6-info-asymmetry]] 在全市场 +
中证 1000/800/500/创业板上做完整因子实证：
- 全市场周度调仓 IC 0.063，信息比 1.18，多头超额年化 +19.07%
- 中证 500 多空对冲信息比 **1.76**（最佳池）
- 与 BARRA 因子相关性低 → 可独立纳入多因子体系

三层信号**机制独立**：
- 九转看的是**价格自身的结构**（连续 N 日未突破）
- 低延时看的是**价格变化率**（斜率开始拉平）
- VPIN 看的是**驱动价格的资金性质**（知情 vs 公共）

理论上三层任意一层先报警都有意义；多层同时报警时确信度最高。

## 信号链时序（理想情况）

```
t-N : VPIN 分位开始抬升 (微观结构警示，知情资金派发开始)
   ↓
t-K : 低延时趋势斜率开始拉平 (价格变化率层面失速)
   ↓
t-J : 九转下跌结构计数开始 (价格结构层面动摇)
   ↓
t   : 九转计满到 N=9 (确认衰竭)
   ↓
t+M : 价格实际回撤 (传统动量信号才察觉)
```

→ **VPIN 在最左、九转在最右**，传统价格信号（如 STR_z reversal）甚至在九转之后。

## 决策框架建议（V25 / ETF_New 集成草案）

### Phase 1：单层叠加（先做）
对 top-1 持仓标的每日计算九转计数 + 低延时趋势斜率。
- 九转计数 ≥ 7（接近触发但未到 9）→ 减仓 30%
- 低延时斜率 < 0 且 |斜率| 加速 → 减仓 50%
- 两层同时报警 → 清仓动量持仓，切防御篮子

### Phase 2：加微观结构层
在 [[ETF_New]] 数据基础上，把 1 分钟 ETF 数据接入计算 VPIN：
- VPIN 分位 > 0.90 → 加入"风险升高"watch list
- VPIN 分位 > 0.95 → 强制减仓 50%
- VPIN > 0.95 + 任意一层（九转/低延时）报警 → 清仓

### Phase 3：与现有机制融合
- 替代 / 补充 dual-window short=10 confluence（短窗口 m=10 本质上是粗糙的"趋势失速检测"，VPIN 更精细）
- 替代 / 补充 STR_z entry tilt（STR 是"事后反弹买回"信号，对 2025 趋势市无效；衰竭检测才是真正的止盈机制）
- 与 max_score cap 互斥（cap 是事前限制，衰竭检测是事中触发；用衰竭检测可拿掉过严的 max cap）
