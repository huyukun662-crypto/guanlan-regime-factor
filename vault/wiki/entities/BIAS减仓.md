# BIAS 减仓

> [[EarlETF]] 在多资产 ETF 动量策略中提炼的实操规则 —— **BIAS 高位减仓 + 温和上涨长持**，与 [[FIP效应]] 的学术发现高度一致

## 一句话定义

把 [[均线偏离度]]（BIAS / 乖离率）作为**事后风控信号**：BIAS 飙升的 ETF（暴涨 → [[信息离散度ID]] 高 → FIP 离散信息组）主动减仓；BIAS 温和的 ETF（慢涨 → ID 低 → FIP 连续信息组）继续持有甚至加仓。

## 核心逻辑

### 直接表述（作者原文）

> "我做 ETF 动量策略这些年，慢慢形成了一个习惯：BIAS 过高的品种，减仓；温和上涨的品种，长期持有。这不是什么深思熟虑的理论推导，更像是被市场教训出来的本能——暴涨的东西拿不住，不是因为你心态差，而是因为它本身就不持久。"

### 学术支撑（[[FIP效应]]）

| 涨法类型 | BIAS 表现 | FIP ID | 6 月动量 | 持续性 |
|---|---|---|---|---|
| 慢涨 | 温和 | 低（连续） | **8.86%** | 8 月 |
| 暴涨 | 飙升 | 高（离散） | 2.91% | 3 月 |

**慢涨的动量是暴涨的三倍** —— 反向证明：BIAS 飙升的品种"涨幅集中、动量短、易反转"。

## 与 [[RSRS]] 的对比

| 维度 | [[RSRS]] | [[BIAS减仓]] |
|---|---|---|
| 思路 | 通过相关系数调整找慢涨 | 通过偏离阈值识别暴涨 |
| 时点 | **事前筛选**（买之前过滤） | **事后干预**（持有后预警） |
| 信号方向 | 找信号买 | 找信号卖 |
| 与 FIP 的关系 | 选连续信息组（ID 低） | 主动剔除离散信息组（ID 高） |
| 起源 | 光大金工 2017 | 作者 2025-2026 摸索 |

> "事先永远无法预测一波涨势是温和还是凶猛，所以 BIAS 减仓是事后干预——水温升得太快的时候，该跳还是要跳的。"

## 操作规则（基于 [[均线偏离度]] 减法版）

> 减法版公式：`BIAS = ln(close) - ln(EMA20)`，× 100% 后近似偏离百分比

| BIAS 区间 | 行动 | 依据 |
|---|---|---|
| < 0 | 退出 | 趋势已弱（[[刘晨明]]：< -5% 离场） |
| 0% ~ 5% | 观察 / 持有 | 趋势正常 |
| 5% ~ 15% | **持有 / 加仓**（理想区间） | 慢涨段，FIP 连续信息组 |
| 15% ~ 20% | **减半仓**（警示） | 进入离散信息边界 |
| > 20% | **清仓 / 大幅减** | FIP 离散信息组，动量易衰竭 |

> 注：阈值参考 [[2026-05-05-gf-strategy-residential-entry-thermometer]]（[[刘晨明]] 减法版）和作者经验，**实际阈值需在 V25 ETF 池子上回测调优**。

## 实战案例（作者举例）

### 黄金 ETF 的转变（2023-2025）

| 阶段 | 特征 | FIP 解读 |
|---|---|---|
| 2023Q4-2024 | 关注度低、走势平稳 | 连续信息组，FIP 增强 |
| 2025H2 | 社交媒体热议、暴涨暴跌 | 转为离散信息组，动量衰竭 |

> "无论是持有还是动量轮动的体验，都变得糟糕了。"

### 创业板 / 科创 50（2024-07~08）

- BIAS 飙升后短期暴涨可观
- 但"那笔钱很难拿住"——涨幅在时间轴上分布不均匀，回撤体验糟糕

## 在 [[ETF轮动与交易策略]] 4 层框架中的位置

```
[1] 候选池
[2] 横向打分
[3] 组合构建
[4] 择时 + 风控   ← BIAS 减仓 ⭐ 反向版的 [[趋势确认过滤]]
```

- [[趋势确认过滤]]：**正向**——温和上涨才上车
- BIAS 减仓：**反向**——暴涨过头就下车
- 两者互为镜像，构成完整的 4 层第 4 层风控

## 工程实施

```python
def bias_trim_signal(close: pd.Series, ema_span: int = 20,
                     warn_threshold: float = 0.15,
                     cut_threshold: float = 0.20) -> str:
    """
    BIAS 减仓信号（减法版）。
    
    Returns
    -------
    str : 'HOLD' / 'WARN' / 'CUT'
    """
    log_close = np.log(close)
    ema = log_close.ewm(span=ema_span, adjust=False).mean()
    bias = (log_close - ema).iloc[-1]
    if bias > cut_threshold:
        return 'CUT'
    if bias > warn_threshold:
        return 'WARN'
    return 'HOLD'
```

## 局限与注意

- 阈值（5% / 15% / 20%）**因品种而异**——价值股长期在 [-5%, 5%] 震荡，本规则会失效
- 不适用于趋势性弱的品种（如中证白酒）
- BIAS 飙升的品种**未来不一定不涨**，但"那笔钱很难拿住"
- BIAS 是**事后**信号 —— 已经吃到一段动量，但放弃了顶部那部分

## V25 集成 Hook

### 优先级 S（直接可测）

1. 在 V25 调仓逻辑加 BIAS 减仓规则：`bias > 0.20` → 强制减半
2. 与现有动量排序合成：动量 Top + BIAS 警戒 = 触发部分平仓

### 优先级 A（升级）

3. **BIAS × ID 双因子矩阵**：
   - BIAS 低 + ID 低 = **重仓**
   - BIAS 高 + ID 高 = **清仓**
   - BIAS 高 + ID 低 = 加速末段，**减半仓**
   - BIAS 低 + ID 高 = 暴跌后修复，**观察**

### 优先级 B（参数化）

4. 阈值按 ETF 历史波动率自适应：`threshold = mean(BIAS) + k × std(BIAS)`
