# FIP 效应（Frog in the Pan）

> 温水煮青蛙效应 —— 投资者对**连续小幅信息**反应不足，对**离散大幅信息**反应充分；前者带来更强、更持久、且不反转的动量

## 一句话定义

[[Da-Gurun-Warachka 2014]]（*Journal of Finance* 69(5)）提出：**信息以连续 vs 离散方式到达，对股价的累积冲击相同时，市场反应程度却显著不同**——连续信息（慢涨）后续动量是离散信息（暴涨）的 **三倍**，且持续 **8 个月不反转**。

## 命名隐喻

> 把青蛙扔进沸水 → 立刻跳出（市场对暴涨的充分定价）
> 把青蛙放进冷水慢慢加热 → 感知不到危险而丧命（市场对慢涨的系统性忽视）

## 核心实证（1976-2007 美股）

| 指标 | 连续信息组（最低 ID 五分位） | 离散信息组（最高 ID 五分位） |
|---|---|---|
| 6 月动量收益 | **8.86%** | 2.91% |
| 差值 | **+5.95%** | t = 5.13 |
| 动量持续期 | **8 个月** | 3 个月 |
| 三年累计三因子 alpha | **+11.77%**（t=2.49） | 负 alpha |
| 后续行情属性 | **不反转**，是低估的持续修正 | 过度反应回调 |

## 衡量工具：[[信息离散度ID]]

```
ID = sign(PRET) × (%neg − %pos)
```

- PRET：过去 12 个月（跳过最近 1 个月）累积收益
- %pos / %neg：形成期内正/负收益天数占比
- ID → -1：连续信息（慢涨）
- ID → +1：离散信息（暴涨）
- **自相关仅 0.034**——衡量"信息到达方式"而非"股票类型"

## 行为金融机制：[[有限注意力]]

- 投资者注意力有下限阈值
- 单次信息冲击不够大 → 不触发关注 → 系统性低估
- 单次信息冲击够大 → 媒体/分析师/社交网络蜂拥 → 充分（甚至过度）反应

**强化条件**（论文 Table 3）：
- 机构持股越分散 → FIP 越强
- 媒体覆盖越少 → FIP 越强

## 排除处置效应（Disposition Effect）

| 论据 | 关键数据 |
|---|---|
| RC（Return Consistency）只能解释赢家延续；ID 对赢家输家**都**有效 | — |
| 分析师盈利预测误差在连续信息后更大 | t = -2.19 |
| Fama-MacBeth：PRET·ID 交互项稳定显著 | t = -10.39 |

分析师不存在处置效应，但同样受 FIP 影响 → 处置效应不能解释 FIP。

## 订单流证据

连续信息组的过去赢家在建仓后三个月持续净买入：3.13 → 2.43 → 3.30；
连续信息组的过去输家持续净卖出。

> 投资者确实在"补作业"——这就是为什么慢涨拿得住、暴涨拿不住。

## 与本土实践的桥梁

### 与 [[均线偏离度]] / [[BIAS减仓]] 的关系

- BIAS 高 = 时间窗口内涨幅集中 → 大概率是离散信息
- BIAS 低 + 价格创新高 = 慢涨累积 → 大概率是连续信息
- **作者经验：BIAS 飙升后减仓 ≈ 主动规避 FIP 离散信息组**

### 与 [[RSRS]] 的关系

| 方法 | FIP 视角 | 时点 |
|---|---|---|
| [[RSRS]] | 高低价相关系数大 → 高低同步 → 慢涨 → 选连续信息组 | **事前筛选** |
| [[BIAS减仓]] | BIAS 飙升 → 离散信息 → 主动减仓 | **事后干预** |

两者**不矛盾，可叠加**。

### 与 [[ETF轮动与交易策略]] 的关系

- A 股 ETF 多数品种**关注度极低**（豆粕 ETF / 行业 ETF 等）→ 论文里的 FIP 强化条件天然成立
- ID 可作为 [[ETF动量排序]] 的"涨法过滤层"
- BIAS 减仓属于 [[趋势确认过滤]] 的反向版本（"涨太狠 → 出"）

## A 股 / ETF 适配性 (INFERRED)

✅ **理论上 FIP 在 A 股 ETF 上更强**：
- A 股 ETF 数百只，多数没媒体覆盖
- ETF 持有人主要是散户和量化基金，不存在主动盯盘的"大股东"
- 完美匹配论文里 "机构分散 + 媒体少 → FIP 增强" 的条件

⚠ **A 股 ETF 适配挑战**：
- ETF 跟踪误差 → ID 信号被误差噪声扭曲
- 流动性差异巨大 → 部分 ETF 的"%pos %neg" 受成交量虚增影响
- 论文样本是 1976-2007 美股个股 → 需 A 股 ETF 独立回测验证

## 实施路径（V25 Hook）

### 优先级 S

```python
def compute_id(returns: pd.Series, lookback: int = 252, skip: int = 21) -> float:
    """
    论文公式：ID = sign(PRET) × (%neg − %pos)
    
    returns: 日收益率序列
    lookback: 形成期长度（默认 252 = 12 个月）
    skip: 跳过最近 N 日（默认 21 = 1 个月）
    """
    window = returns.iloc[-lookback-skip:-skip]
    pret = (1 + window).prod() - 1
    pct_pos = (window > 0).mean()
    pct_neg = (window < 0).mean()
    return np.sign(pret) * (pct_neg - pct_pos)
```

### 优先级 A

- ID × 动量双层排序：先选动量 Top → 再按 ID 取最连续的前 50%
- BIAS 高位减仓：BIAS_减法版 > 15% 减半仓 / > 20% 清仓
- ID + RSRS + BIAS 三合一：RSRS 选 → ID 验证 → BIAS 调
