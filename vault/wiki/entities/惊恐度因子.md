# 惊恐度因子（方正"惊恐"系列）

> 方大证券 2022-12 提出的 [[凸显性因子STR]] A 股化版本 — **守株待兔 + 草木皆兵心理的量化**

## 一句话定义

不直接照搬美股 STR 的 δ-power weight，而是**直接用"惊恐度"σ 加权个股收益率** → 月度 mean + std 等权合成"原始惊恐"因子。基础版上扩展 4 个变形（波动率加剧 / 个人投资者比 / 注意力衰减），共**4 个子因子**。

## 来源

方大证券《显著效应、极端收益扭曲决策权重和"草木皆兵"因子》— 2022-12-13

仓库 PDF：`参考/20221213_方大证券_显著效应、极端收益扭曲决策权重和"草木皆兵"因子.pdf`

## 心理学解读

### "守株待兔"（极端高收益）
- 看到股票暴涨 → 投资者认为再来一次
- 过度买入 → 推高价格
- 未来回落（**反向因子**）

### "草木皆兵"（极端低收益）
- 看到股票暴跌 → 投资者恐惧再亏
- 过度卖出 → 压低价格
- 未来补涨（**反向因子**）

⇒ 极端偏离市场的收益**双向都被高估了的反应力度**。

## 核心公式

### 惊恐度（与 STR 同公式但基准不同）
```
惊恐度_{i,d} = |r_{i,d} - r_{HS300,d}| / (|r_{i,d}| + |r_{HS300,d}| + 0.1)
```

**关键差异**（vs 美股 STR）：
- 用**沪深 300 收益率**作市场基准（替代截面平均）
- 不再做 δ-power weight，**直接用惊恐度乘收益**

### 加权决策分
```
weighted_score_{i,d} = 惊恐度_{i,d} × r_{i,d}
```

### 月度合成
```
惊恐收益_i = mean(weighted_score, 20)    # 替代 20 日收益率因子
惊恐波动_i = std(weighted_score, 20)     # 替代 20 日波动率因子
原始惊恐_i = (惊恐收益 + 惊恐波动) / 2   # 等权合成
```

## 4 个子因子变形

| 因子 | 额外权重项 | 直觉 |
|---|---|---|
| **原始惊恐**（v0） | 仅惊恐度 × 收益率 | 基础版 |
| **波动率加剧-惊恐** | × 1 分钟频率波动率 | 高频 vol 强化"凸显" |
| **个人投资者交易比-惊恐** | × 个人投资者占比 | 散户更易被极端收益吸引 |
| **注意力衰减-惊恐** | × (今日 σ - 过去 2 日 σ 均值)正部分 | 边际新增注意力 |

### 个人投资者占比（关键 proxy）
> 单笔成交金额 < 4 万元的交易 = 个人投资者交易（参考 Wind 资金流指标）

```
个人投资者交易比 = (个人买入金额均值 + 个人卖出金额均值) / 当日总成交金额
```

### 注意力衰减（捕捉"边际新增"）
```
衰减惊恐度 = max(0, σ_t - mean(σ_{t-1}, σ_{t-2}))
# 仅保留今日 σ 大于过去 2 日 σ 均值的样本
# 覆盖率较低 → 月度计算只要 5 条数据就能算
```

## Python 实现（[[hugo2046]]）

```python
# 用沪深 300 作基准
sigma  = pct_chg.pipe(calc_sigma, bench=hs300_returns)

# 加权决策分
weighted = sigma.mul(pct_chg)

# 惊恐收益 + 惊恐波动
avg_score = weighted.rolling(20).mean()    # 惊恐收益
std_score = weighted.rolling(20).std()     # 惊恐波动

# 原始惊恐 = 等权合成
terrified_score = (avg_score + std_score) * 0.5
```

## 与 [[凸显性因子STR]] / [[STV凸显性量价因子]] 的对比

| 维度 | v1 美股 STR | **v2 惊恐**（本因子）| v3 STV |
|---|---|---|---|
| 基准 | 截面平均 | **HS300** | 截面平均 |
| 权重 | δ^k | 直接 σ | δ^k |
| 月度合成 | cov(ω, r) | **mean + std 等权** | cov(stv_w, r) |
| 子因子数 | 1 | **4** | 1 |
| 适配 A 股 | 弱 | 中 | 强 |

⇒ 惊恐系列**不如 STV 适配 A 股**（未涉及涨跌停），但**子因子数最多 → 信息维度更丰富**。
