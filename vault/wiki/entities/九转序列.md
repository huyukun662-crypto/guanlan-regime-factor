# 九转序列

> 借鉴 **TD Sequential** 理论 — 通过价格结构连续性计数判断**趋势衰竭**并预测拐点

## 一句话定义

若价格连续 N 日处于下跌状态（如收盘价持续低于前几日）但过程中**未创新低** → 说明市场已有趋弱迹象，趋势或将终结。设定"收盘价 < 前 4 日收盘价"为下跌结构标准，连续计数到阈值 N（典型 N=5 或 N=9）触发反转信号。

## 历史溯源

- 原始理论：**TD Sequential**（Tom DeMark 提出）
- 在 A 股 / 利率期货中常被称为"九转"或"神奇 9 转"
- 经典 N=9 → "九转底" / "九转顶"

## 信号生成机制

1. **下跌结构计数**：每日检测"收盘价 < 4 日前收盘价"
2. **连续累计**：连续满足条件的日数即为"九转计数"
3. **触发反转信号**：达到阈值（N=9 或 N=5）时
4. **量能确认**（增强）：要求反转结构伴随量能验证
5. **趋势过滤**（增强）：仅在主趋势反向时采纳信号 → 避免在趋势强势背景下博弈逆势

## 优势

- **拐点识别能力强** — 信号清晰
- **触发频率适中** — 既不太稀疏也不太密集
- **适用场景明确**：
  - 趋势尾部减仓提示
  - 震荡区内的反向建仓依据

## 劣势

- **信号稀疏，略显保守** — 趋势初期可能迟滞
- 需配合趋势类模块使用以形成**信号闭环**

## 在 [[多信号策略]] 中的角色

- 三层结构中的**结构衰竭层**（与趋势识别 / 节奏控制层互补）
- 与趋势类信号（[[低延时趋势]]、[[单/双均线策略]]）配合 → 闭环

## V25 复现 Hook

```python
def td_sequential(close, n=9, lookback=4):
    """简单 TD Sequential 计数"""
    bear_count = 0
    signals = []
    for i, c in enumerate(close):
        if i < lookback:
            signals.append(0)
            continue
        if c < close[i-lookback]:
            bear_count += 1
        else:
            bear_count = 0
        if bear_count >= n:
            signals.append(1)  # 反转信号
            bear_count = 0
        else:
            signals.append(0)
    return signals
```

**优先级最高的子策略**（5 个里实现最简单 + 标准化最好）。

## 在动量衰竭早期识别框架中的角色

九转是 [[动量衰竭早期识别]] 三层框架中的**价格结构层**（最右端、最确认）：
- 在 [[VPIN]]（微观结构层）和 [[低延时趋势]] 斜率拉平（趋势导数层）之后才触发
- 信号清晰但稀疏，作为"最后确认"使用而非提前预警
- 适合作为 ETF 动量持仓的"清仓触发线"（计满 N=9 即清仓），而 VPIN/低延时用于"分批减仓"
