# 拓扑结构选股（图论 / 相关性网络）

把全市场当成一张**图（Graph）**：节点=标的，边=价格变动相关性。用图论工具刻画"谁和谁同涨同跌、谁是枢纽、传染怎么走"。

## 核心构件

| 概念 | 定义 | 用途 |
|---|---|---|
| 相关性矩阵 | 标的两两收益相关系数 | 建图的边权 |
| 最小生成树 MST | 割弱边、保留连通全图的最强边骨架 | 给上千万条边的稠密图"剪枝"，得清晰骨架 |
| 连通分量 / 社区 | 强连接成团的标的簇 | 识别"同质集群"（如半导体+芯片设备+AI 一起动） |
| 中心度 Centrality | 节点连接度 / 特征向量重要性 | 找"枢纽标的"（跨板块交汇、传染源） |

## 三种用法（及其有效性判断）

1. **传染风控（枢纽异动→预判雪崩→清仓）** — 本质是**市场择时 / regime overlay**。
2. **中心度当选股因子（买枢纽 / 桥梁标的）** — 截面 alpha 尝试。
3. **相关性聚类做分散约束（同簇限仓）** — 风险约束，对接 [[低相关组合]] / [[ETF候选池构建]] 的聚类去重。

## 不同素材中的观点

- [[2026-05-23-graph-topology-astock-network]]（科普）：强调用法①传染风控"量化总能暴跌前逃生"——**修辞性强、无回测证据**。
- [[2026-05-07-jq-etf-cross-section-thinking]]（截面思维）：把"相关性聚类去重"放在**第 1 层候选池**与**第 3 层低相关组合**，是更工程化的同源思想。

## ETF 7.0 实测结论（重要）

在 **114 ETF** 池上实测对接：
- 用法①（拓扑择时风控）**先验强负**——与 ER gate / RSRS / RSRS+过滤 一同属"市场择时叠加"，已在该池**多次否决**（择时砍反弹、降不了 −18.5% 回撤）。
- 用法③（**相关性聚类上限**）是**唯一契合**方向：滚动相关性图→连通分量→约束突破腿"同簇 ≤N 只"降共振回撤，对齐 ETF2.0"类别上限有效"老结论。PoC：`ETF 7.0/cluster_cap_test.py`。
- 注意：图论方法为**大截面（数千股）**设计；ETF 已是成分股篮子，相关性由少数宏观因子主导，**网络信息量大幅下降**。
