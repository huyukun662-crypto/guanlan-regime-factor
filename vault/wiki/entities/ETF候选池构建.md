# ETF 候选池构建

> ETF 截面框架的**第 1 层**（最易被忽略但最关键） — "**先别急着调因子，先把池子做干净**"

## 一句话定义

ETF 策略中"打分排序"之前的**入口净化**：剔除流动性差、成立期短、同质化高、相关性高的 ETF，把池子从"原始市场所有 ETF" 收敛到 "可比较 / 可买卖 / 不重复"的工作集。

## 6 类常见过滤

| 过滤维度 | 标准 | 解决问题 |
|---|---|---|
| **成立时间** | 上市 < 6 月 / 1 年的剔除 | 避免新股噪音 + 历史数据不足 |
| **流动性** | 日均成交额 < N 亿剔除 | 避免买不进 / 卖不出 |
| **同质化** | 跟踪同一指数的 ETF 只保留 1 只 | 避免重复持仓 |
| **走势相似** | 用聚类或相关系数阈值合并 | 避免"表面分散" |
| **资产分类** | 按主题 / 跨境 / 商品 / 债券分组 | 让横向比较更公平 |
| **黑名单** | 即将退市 / 变更基准 / 大额赎回 | 避免突发风险 |

## 实施手段

### A. 流动性过滤
```python
# 例：剔除最近 N 日均成交额 < 0.5 亿的 ETF
universe = [etf for etf in pool if vol_avg(etf, 60) >= 5e7]
```

### B. 同质化聚类（建议）
```python
# 用相关矩阵 + 阈值聚类
corr = etf_returns.corr()
clusters = []
for etf in universe:
    matched = [c for c in clusters if any(corr[etf][m] > 0.95 for m in c)]
    if matched: matched[0].append(etf)
    else: clusters.append([etf])
# 每个聚类只保留代表性 ETF（如成交量最大的）
```

### C. 资产分类树
```
ETF 池
├── 宽基（沪深300 / 上证50 / 中证500 / 中证1000）
├── 行业（大消费 / 医药 / 半导体 / 新能源 / ...）
├── 主题（红利 / 央企 / 央国企改革 / ...）
├── 跨境（纳指 / 标普 / 恒生 / ...）
├── 商品（黄金 / 白银 / 原油 / ...）
└── 债券（利率债 / 信用债 / ...）
```
