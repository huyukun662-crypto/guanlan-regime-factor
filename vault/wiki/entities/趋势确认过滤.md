# 趋势确认过滤

> ETF 截面框架的**第 4 层** — 排完名后再确认趋势，避免"虽然是 Top 但已经不舒服"

## 一句话定义

[[ETF动量排序]] 后**多加一层趋势 / 状态过滤**：解决"它虽然在这批 ETF 里最强，但我现在买进去是不是太晚了？" — 把"先算分谁第一买谁"升级为"**先算分 → 再确认 → 通过才上车**"。

## 解决的核心问题

横向比较只能告诉你**相对强弱**，回答不了：

> **"相对强，但可能已经不舒服"**

这是 ETF 轮动从"追涨切换器"升级为"成熟系统"的关键点。

## 6 种常见过滤层

| 过滤层 | 逻辑 |
|---|---|
| [[RSRS]]（阻力支撑相对强度）| 高低价 OLS 斜率 + 标准化 |
| **均线站上**（如 MA20）| 价格 > MA20 才上车 |
| **BBI / 多均线状态** | 多条均线多头排列 |
| **BOLL 压缩后突破** | 波动率收敛后释放 |
| **成交量异常过滤** | 缩量上涨 = 不健康 |
| **日内大跌排除** | 当日跌幅 > X% 暂停 |

## 与 [[ETF动量排序]] 的结构升级

### 旧结构（仅排序）
```
打分 → 排序 → 直接买 Top
```
- 缺点：无视当前买点的"舒适度"
- 高换手 / 易追在山顶

### 新结构（排序 + 确认）
```
打分 → 排序 → 取 Top M → 趋势确认 → 通过的才上车
```
- 优点：过滤掉"强但已经不舒服"
- 减换手 / 改善买点

## 实施细节：避免过度叠加

> "**只做真正必要的事：大环境不过关时收缩风险，个别标的失效时退出，而不是让过滤器层层叠叠把原始逻辑挤没。**"
> — 来自 [[2026-05-07-jq-etf-cross-section-thinking]]

⚠ **典型错误**：在第 4 层叠加 5-10 个过滤器 → 信号过滤到几乎不触发 → 错过整段行情

⇒ **建议**：选 1-2 个最关键的过滤层（如 MA20 + 大盘 regime），其他都是噪音放大器

## 与已有素材的关联

### 与 [[价格趋势过滤]]（OpenAlphas RSRS 改进）
- OpenAlphas RSRS 加入 "MA20 上升趋势"过滤 → MDD 50%+ → 20%
- 这是趋势确认过滤的**实战范本**

### 与 [[成交量相关性过滤]]（OpenAlphas RSRS 改进）
- 用 10 日成交量与 RSRS 修正标准分相关性过滤
- 也属于趋势确认过滤家族

### 与 [[均线偏离度]]（广发刘晨明）
- 5%-15% 是"良性"区间 → "良性"才上车 ≈ 趋势确认
- 阈值过滤本身就是一种趋势确认

### 与 [[多信号策略]]（浙商证券）
- 浙商的"任意 2 信号触发"是同一思想
- 但浙商是**单标的多信号**，本概念是**多标的中的过滤**
