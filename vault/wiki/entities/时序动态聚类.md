# 时序动态聚类

> Rolling K-means / Time-series Dynamic Clustering — **每个决策窗口重新做 K-means**，让聚类随市场状态演变

## 一句话定义

把传统 K-means **静态聚类**改造为**滚动窗口聚类** — 每个决策点（如月度）用前 L 天数据重新拟合 K-means → 得到当下最优聚类 → 用于强化学习决策。**显著优于一次性拟合的静态聚类**。

## 解决的问题

### 静态聚类的失效
- 传统：在初始训练窗口拟合 K-means → 后续永久使用
- 问题：**这一时期最优聚类下一时期未必最优**
- 例子：市场状态从"成长股主导"切换到"价值股主导"时，原聚类边界全部错位

### 动态聚类的价值
- 每月重新做 K-means → 聚类边界随市场调整
- 智能体在**新聚类**上做决策 → 自适应市场状态切换

## 实施步骤（[[Q-A3C²]] 中）

```python
# 每个月开始时:
for t in 月度决策点:
    # 1. 提取最近 L 天的特征矩阵
    features = stack([
        cumret_5d(stocks, t),      # 5 日累计收益
        cumret_20d(stocks, t),     # 20 日累计收益
        vol_20d(stocks, t)         # 20 日波动率
    ], axis=1)

    # 2. 重新拟合 K-means
    kmeans = KMeans(K).fit(features)

    # 3. 把每只股票分配到聚类
    cluster_map = {stock: kmeans.predict(feat) for stock, feat in zip(...)}

    # 4. 计算聚类级别状态
    state = compute_cluster_state(cluster_map, ...)

    # 5. 智能体决策
    action = agent.policy(state)
    # 投资聚类 action 中的所有股票（等权）
```

## 为什么 K = 4？

[[Q-A3C²]] 论文用 K=4 个聚类（"4K + 2 维状态"中的 4）：
- K 太小 → 聚类粗 → 浪费信息
- K 太大 → 每个聚类样本少 → 估计不稳定 + 状态空间爆炸
- 4 是个**经验平衡点**（论文未做敏感性测试）

## 与已有素材的关联

### 与 [[2026-05-05-qpb-industry-pricevolume-etf-rotation]]（华西因子失效）
- 华西方案是**静态多因子**（在 ETF 上失效）
- 本方法是**动态聚类**（部分回应了那个反例）
- 启示：**ETF 池上需要动态适应方法**

### 与 [[底部双因子识别]]（OpenAlphas）
- 底部双因子是**手工 + 阈值**判断市场状态（MDD + PE）
- 时序动态聚类是**自动 + 数据驱动**判断市场状态
- 同一思路的 ML 化版本

### 与 [[NPEB法则]]（Lai 团队）
- NPEB 用 Bootstrap 处理参数不确定性
- 时序动态聚类用 rolling 处理状态不确定性
- 都属"承认市场不平稳，动态适应"流派

## 优势 / 劣势

✅ **自适应市场状态切换**
✅ 工程实现简单（K-means 是标准库）
✅ 与 RL / NN 框架天然兼容

⚠ **K 选择没有原则性方法**（试错）
⚠ K-means 假设球形聚类（可改为 Gaussian Mixture / Spectral）
⚠ **聚类突变**：相邻两月的聚类完全不同时，模型难以学到稳定策略
⚠ **计算成本**比静态高 12 倍（每月一次 vs 一次到底）

## 改进方向（来自论文消融）

- 滚动 + 没有 RL：贪心选上月最高回报的聚类（启发式 baseline，效果不如本方案）
- 静态聚类 + RL：性能下降明显
- ⇒ **滚动 + RL** 是性能关键
