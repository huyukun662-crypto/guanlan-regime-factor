# 贝叶斯超参优化

> Bayesian Hyperparameter Optimization — 通过先验分布建模目标函数、迭代选择期望提升最大的参数组合的智能搜索方法

## 一句话定义

贝叶斯超参优化通过对目标函数（验证集指标）建立**代理模型（surrogate）**，每一轮根据**采集函数（acquisition function）**选择期望提升最大的参数组合，相比网格搜索和随机搜索，**在高维参数空间中收敛速度快几个数量级**。

## 三种主流超参优化方法对比

| 方法 | 原理 | 适用场景 | 缺陷 |
|---|---|---|---|
| **网格搜索** | 遍历所有参数组合 | 2-3 个超参，离散值 | 高维参数空间几乎不可用，计算量爆炸 |
| **随机搜索** | 随机采样参数组合 | 中维超参 | 不利用历史信息，效率有限 |
| **贝叶斯优化** | 用代理模型 + 采集函数迭代选择 | 高维 + 连续 + 训练耗时 | 实现复杂，并行性差 |

## 在量化投资场景中的优势

- **避免手动调参的盲目性**：传统手调超参容易陷入局部最优 / 主观偏见
- **大幅提升泛化能力**：通过验证集 IC 作为目标函数，超参直接面向"样本外稳定性"
- **降低过拟合风险**：通过代理模型的不确定性估计避免过早收敛到训练集最优

## 工程实现：[[Optuna]]

- Python 库 `optuna`
- 默认采集函数：TPE（Tree-structured Parzen Estimator）
- 简单 API：`study.optimize(objective, n_trials=50)`
- 内置剪枝（pruning）机制：早停明显劣于历史最优的试验

## 标准用法（量化场景）

```python
import optuna

def objective(trial):
    params = {
        'min_data_in_leaf': trial.suggest_int('min_data_in_leaf', 20, 200),
        'feature_fraction': trial.suggest_float('feature_fraction', 0.5, 1.0),
        'bagging_fraction': trial.suggest_float('bagging_fraction', 0.5, 1.0),
        'lambda_l1': trial.suggest_float('lambda_l1', 0, 10, log=False),
        'lambda_l2': trial.suggest_float('lambda_l2', 0, 10, log=False),
        # ... 其他超参
    }
    model = lgb.train(params, train_data)
    pred = model.predict(val_X)
    ic = spearmanr(pred, val_y).correlation
    return ic  # 最大化验证集 IC

study = optuna.create_study(direction='maximize')
study.optimize(objective, n_trials=50)
best_params = study.best_params
```
