# 时序划分

> 严格按时间序列将数据划分为训练 / 验证 / 测试三段，**禁止用未来数据训练模型**

## 一句话定义

量化金融中**最重要的工程纪律之一**：数据集必须**严格按时间序列**划分（早期数据训练 / 中期数据验证 / 后期数据测试），训练时**不使用任何未来数据**，避免**数据泄露 / 前视偏差**。

## 与传统 ML 划分的区别

| 划分方式 | 训练数据 | 测试数据 | 是否泄露 |
|---|---|---|---|
| 随机切分（K-fold） | 随机 50% | 随机 50% | **会泄露**（用未来训练） |
| **时序划分** | 早期 70% | 后期 30% | 不泄露 |
| Walk-forward（滚动验证） | 滚动早期窗口 | 紧邻后续窗口 | 不泄露 + 多次验证 |

## 标准三段切分（OpenAlphas 用法）

```
2019-01-01 ~ 2023-06-30   → 训练集（最优超参在此训练）
2023-07-01 ~ 2024-06-30   → 验证集（IC 评估超参 / 早停）
2024-07-01 ~ 2025-01-01   → 测试集（最终样本外回测）
```

最优超参确定后，可以用 **训练集 + 验证集** 一起 retrain 最终模型，用测试集评 IS/OOS 一致性。

## 数据泄露的常见陷阱

1. **特征构造时用了未来数据**：例如计算因子时用了 `groupby().mean()` 整列均值（含未来）
2. **目标标准化时用了全样本统计量**：标签 Z-score 时 `mean / std` 含测试期
3. **Hyperopt 用全样本**：超参优化目标看到了测试期 → 隐式过拟合
4. **数据集对齐错位**：`shift(-1)` 写错变 `shift(0)` 直接用了同期未来收益

正确写法（V25 项目纪律）：
```python
# 滚动 / 截面统计只用当前及之前数据
expanding_max_correct = nav.loc[:today].max()           # ✓
expanding_max_wrong   = nav.expanding().max()           # ✗ 全序列含未来
```
