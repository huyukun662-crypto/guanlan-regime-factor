# 标准差过滤器

> 考夫曼 AMA 完整交易系统的"变速箱"组件 — **用过去 N 期 AMA 变化的标准差作为信号触发的门槛**，过滤"假突破 / 毛刺"信号。

## 一句话定义

**只有当本期均线（或价格）的变化幅度足够大（超过近期变化波动的 k 倍标准差）时，才允许触发交易信号**，从而剔除震荡市中的微小毛刺。

## 公式（考夫曼原版方向）

```
ΔAMA_t = AMA_t - AMA_{t-1}
σ_n    = std(ΔAMA, lookback=n)

if abs(ΔAMA_t) > k * σ_n:
    signal_active = True
else:
    signal_active = False    # 被过滤
```

> 原文未给出具体的 `k` 与 `n` 取值 <!-- confidence: AMBIGUOUS -->
> 业界常见默认：`n = 10`、`k ∈ [1.0, 2.0]`，但需独立寻优。

## 为什么需要这个过滤器

裸 KAMA 在两类情景下仍会发假信号：

1. **平台震荡 + 微小漂移**：KAMA 几乎水平但有微小斜率变化，被快线穿越触发金叉死叉
2. **单根异常 K 线（毛刺）**：单日跳空导致 AMA 短暂跳动一下，过两天又走回去

加上 σ 过滤后，"足够大才算数"的判断把这些噪声拒之门外。

## 通用版本（非 KAMA 专属）

这个机制其实是**信号触发上加波动率自适应阈值**的通用范式，可以套到多种信号：

| 应用场景 | 信号变量 | σ 度量 |
|---|---|---|
| KAMA 双均线 | AMA 差值 | std(ΔAMA) |
| MA 突破 | 价格 - MA | std(残差) |
| Bollinger 中轨突破 | 价格变化 | std(returns) |
| RSI 阈值穿越 | RSI 变化 | std(ΔRSI) |

**核心思想**：阈值不是常数，而是随近期波动率呼吸——市场越平静，阈值越严；市场越剧烈，阈值越宽。

## 与本知识库已有信号过滤思路的关系

- 与 [[价格趋势过滤]]（MA20 滤波）：MA 过滤是"位置过滤"（在 MA 上方才允许多头信号）；σ 过滤是"幅度过滤"（变化够大才允许信号）— 两者正交，可叠加
- 与 [[3σ去极值]]：3σ 是因子值预处理阶段去极值，目标是平滑因子分布；本过滤器是信号阶段去毛刺，目标是减少交易次数
- 与 [[FIP效应]] / 趋势光滑度：FIP 是事后判断趋势质量；σ 过滤是事前提高信号阈值 <!-- confidence: INFERRED -->

## 在 V25 ETF 项目中的潜在应用

可作为 grid_search 的**一维新参数**（A 档候选机制）：

```
σ_filter_k ∈ [0.0, 0.5, 1.0, 1.5, 2.0, 2.5]
σ_filter_n ∈ [5, 10, 20]
```

- `k = 0` 退化为不过滤（基准）
- `k = 2` 表示需要"近 n 期变化中较罕见的大幅波动"才算信号
- 期望效果：降低换手率、减小回撤、可能略损 Sharpe（典型 Pareto 权衡）

## 局限性

- 参数 `k` 选择需独立寻优，没有"放之四海皆准"的默认值 <!-- confidence: EXTRACTED -->
- 牛市初期的"温和上涨"可能被过滤掉（变化幅度温和，但应入场）
- 与 [[均线偏离度]] BIAS 阈值过滤思路类似，需检查二者是否产生信号叠加冗余 <!-- confidence: INFERRED -->
