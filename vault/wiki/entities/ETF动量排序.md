# ETF 动量排序

> ETF 截面框架的**第 2 层主流方法** — 给一篮子 ETF 横向打分排序，买最强的几只

## 一句话定义

最常见的 ETF 轮动起点：**给定 ETF 池 → 算各自的动量分数 → 按分数排序 → 买 Top 1 或 Top N**。简单、透明、易扩展，但**换手率会很高**（不是趋势没了而是排名变了）。

## 5 种常见动量分数写法

| 公式 | 特点 |
|---|---|
| **近 N 日累计收益** | 最简单 |
| **价格 / 对数价格的线性回归斜率** | 平滑但滞后 |
| **年化收益 × R²** | "动量 + 趋势确认"合一 |
| **相对中期均线的乖离趋势** | 与 [[均线偏离度]] 同源 |
| **带近期加权的趋势回归分数** | 对最近变化更敏感 |

## 实施模板

```python
import pandas as pd

def momentum_score(prices: pd.DataFrame, lookback: int = 60):
    """
    Top N 动量排序的标准实现
    prices: 列=ETF代码，行=日期
    """
    cum_ret = prices.iloc[-1] / prices.iloc[-lookback] - 1
    return cum_ret.sort_values(ascending=False)

# 使用
ranked = momentum_score(etf_prices, lookback=60)
top_n = ranked.head(3).index.tolist()
```

## 优点

- ✅ **简单**：一行排序就能出结果
- ✅ **透明**：每只 ETF 为什么被选 / 不选都可解释
- ✅ **可扩展**：宽基 / 行业 / 商品 / 跨境 都能横向比
- ✅ **零参数**（动量公式 1 个超参 lookback）

## 副作用：高换手率

> "**只要你是"每天重新排一次名，然后跟着第一名走"，换手率通常不会低。**"
> "**很多时候，不是趋势没了，而是排名变了。**"
> "**不是因子没用，而是截面排序太敏感。**"

### 缓解换手率的方法

| 方法 | 实现 |
|---|---|
| **延长调仓周期** | 月调仓 vs 周调仓 vs 日调仓 |
| **加 buffer 区** | 当前持有的 ETF 排名跌出 Top N+2 才换 |
| **加趋势确认层**（[[趋势确认过滤]]） | 先打分排序 → 再确认 → 才上车 |
| **结合 [[低相关组合]]** | 不光看排名 → 还看互补性 |
| **持仓分散到 Top N** | 不单押 Top 1 |
