# 低相关组合

> ETF 截面框架的**第 3 层** — 把"谁最猛？"问题转变为"**谁和谁别太像？**"的组合构建思路

## 一句话定义

不只在乎选出的 ETF 是不是最强，更在乎选出的几只 ETF **背后的风险驱动是否过于一致**。表面 3 只 ETF 看似分散，实际可能背后是同一种风险的不同壳 → 压力环境一来一起暴跌。**低相关 = 真正的分散。**

## 核心问题转变

| 旧问法 | **新问法（低相关思路）** |
|---|---|
| "**谁最猛？**" | "**谁和谁别太像？**" |
| 选 Top 1 / Top N 强势 | 选**强势集合中相关性最低**的一组 |
| 追涨切换器 | 真正的组合 |

## 三种实施做法

### 做法 1：先剔除高相关，再选强
```
ETF 池 → 计算相关矩阵 → 剔除相关 > 阈值的标的 → 在剩下池里选 Top N
```

### 做法 2：先选强，再选低相关子集（推荐）
```
ETF 池 → 趋势 / 动量过滤选 Top M → 在 Top M 里选**平均相关性最低**的 N 只
```

### 做法 3：低相关 + 趋势同时优化
```
最大化 Σ(动量分_i) - λ × Σ(相关_ij)
```

## 关键认知（来自原文）

> "**收益好的时候，很多 ETF 看起来谁都能涨；压力环境一来，才会发现买的其实是一种风险的不同壳。**"

> "**低相关不一定最亮眼，但通常更稳。**"

⇒ 低相关组合的真正价值在**压力环境**而非顺风期。

## 与 [[ETF动量排序]]（Top 1 / Top N）对比

| 维度 | Top 1 / Top N | **低相关组合（本方法）** |
|---|---|---|
| 选股逻辑 | 排序最强 | 强势 + 低相关 |
| 换手率 | 高（排名敏感）| **较低**（结构性持仓）|
| 顺风期表现 | **更亮眼** | 略逊 |
| 压力期表现 | 易共振下跌 | **更稳** |
| 形态 | 追涨切换器 | **真正的组合** |

## 与 [[ETF候选池构建]] 的关系

- **候选池层**（第 1 层）：删掉**完全重复**的 ETF（如跟踪同一指数的多只 ETF）
- **低相关组合**（第 3 层）：在已经"独立"的 ETF 中进一步选**风险驱动不重叠**的子集

⇒ 两者**解决不同颗粒度的问题**：候选池处理"是否重复"，低相关处理"是否同步"。

## 实施细节

### 相关矩阵估计的稳定性
- 用过去 N 月（推荐 6-12 月）日收益率算相关
- 短窗口 → 噪声大，长窗口 → 错过 regime 变化
- 可用 EWMA 加权平均（近期权重高）

### 阈值选择
- 一般 ρ > 0.85 视为高度相关，可考虑剔除其一
- 严格用 ρ > 0.7

### 平均相关 vs 最大相关
- 平均相关：对**整体分散度**敏感
- 最大相关：对**最差子集**敏感
- 实战常组合用：先看平均，再用最大兜底
