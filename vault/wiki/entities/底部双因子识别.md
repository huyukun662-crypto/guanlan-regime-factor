# 底部双因子识别

> 用 [[上证综指3年滚动MDD]] 与 [[上证综指PE分位]] 的合取（AND）定义市场底部，避免单一指标的误判。

## 定义

```python
df["is_bottom_zone"] = (df["MDD"] >= 0.2) & (df["pe_percentile"] <= 10)
```

只有当 **价格条件** 与 **估值条件** 同时满足，才进入"底部区域"状态。

## 设计哲学

- 单用价格：可能在结构性调整 / 流动性危机中过早触发
- 单用估值：可能在估值长期下行通道中长期触发但无反弹机会
- **AND 合取**：要求两个独立维度同时报警，假阳性率显著下降，但代价是**频次极低**（A 股 20 年只触发了 6 次大底）

## 触发的下游动作

```python
# 底部确认 + SRI > 均衡 → 配价值
df.loc[(is_bottom_zone) & (SRI > SRI_balance), "signal"] = 1
# 底部确认 + SRI <= 均衡 × 1.05 → 配成长
df.loc[(is_bottom_zone) & (SRI <= SRI_balance * 1.05), "signal"] = 2
# 非底部 → 50/50 均衡
df.loc[~is_bottom_zone, "signal"] = 0
```

## V25 应用候选（详见 [[2026-04-23-openalphas-bottom-style-timing]]）

- M2：进入双因子底部区域时，放宽 V25 的 trail_recover 阈值（从 -3% 到 -1%），鼓励早进场
- M3：把 PE_percentile<10% & MDD>=20% 作为强制 gate=ON 条件，避免被 4w drop 误杀切防御
